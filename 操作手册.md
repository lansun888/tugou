# 土狗猎手 (Tugou Hunter) - BSC 链上交易机器人操作手册

## 1. 项目简介
本项目是一个基于 Python (后端) 和 React (前端) 的 BSC (Binance Smart Chain) 链上自动交易机器人。主要功能包括：
- **新币监控**：实时监听 PancakeSwap V2 新创建的交易对。
- **安全分析**：自动检测 Honeypot (貔貅盘)、合约开源状态、持有者分布等安全指标。
- **极速买入**：优化的多节点并行执行和交易预构建技术，实现毫秒级抢购。
- **自动风控**：支持分批止盈、追踪止损、时间止损等多维度风控策略。
- **可视化界面**：提供现代化的 Web 界面进行监控、配置和资产管理。

## 2. 快速启动

### 2.1 环境要求
- **操作系统**：Windows / macOS / Linux
- **Python**：3.10 或更高版本
- **Node.js**：16.0 或更高版本

### 2.2 安装依赖
首次运行前，请确保已安装所有依赖。本项目建议使用 `bsc_bot/venv` 虚拟环境（如果存在）或创建新的虚拟环境：

```bash
# 后端依赖 (Windows)
# 如果使用内置虚拟环境:
.\bsc_bot\venv\Scripts\pip install -r requirements.txt

# 或者使用系统 Python:
# pip install -r requirements.txt

# 前端依赖
cd frontend
npm install
cd ..
```

### 2.3 数据初始化 (可选)
如果您希望在启动前填充一些测试数据（如模拟交易记录、持仓信息等），可以运行以下脚本：

```bash
# 初始化并填充测试数据 (使用虚拟环境 Python)
.\bsc_bot\venv\Scripts\python.exe scripts/init_data.py

# 如果想清空现有数据并重新初始化，请添加 --reset 参数
.\bsc_bot\venv\Scripts\python.exe scripts/init_data.py --reset
```

### 2.4 启动项目
Windows 用户可以直接双击根目录下的 **`start_web.bat`** 脚本，它会自动启动后端 API 和前端页面。

启动后，浏览器会自动打开：
- **Web 界面**：[http://localhost:3000](http://localhost:3000)
- **API 文档**：[http://localhost:8000/docs](http://localhost:8000/docs)

## 3. 功能模块说明

### 3.1 仪表盘 (Dashboard)
- **概览**：显示当前 BNB 余额、今日盈亏、运行时间等核心指标。
- **活跃仓位**：实时展示当前持有的代币及其盈亏状态。
- **性能监控**：显示节点延迟和系统状态。

### 3.2 实时日志 (Logs)
- 类似终端的黑底绿字界面，实时显示机器人的运行日志。
- 支持按 **级别** (INFO/WARNING/ERROR) 和 **模块** (监控/交易/分析) 筛选。
- 关键事件（如发现新币、买入成功、止盈卖出）会有高亮显示。

### 3.3 发现 (Discoveries)
- 列表显示所有监听到的新代币。
- 包含代币名称、合约地址、初始流动性、安全评分等信息。
- 点击代币可查看详细的安全分析报告。

### 3.4 仓位管理 (Positions)
- 管理当前持有的所有代币。
- **操作**：
  - **手动卖出**：点击“卖出”按钮可手动平仓（支持 25%/50%/75%/100% 比例）。
  - **查看详情**：查看买入价格、当前价格、未实现盈亏。

### 3.5 设置 (Settings)
支持在页面上动态修改配置，无需重启服务：
- **交易参数**：
  - **每次买入金额**：设置单笔交易的 BNB 数量。
  - **滑点设置**：调整买入/卖出的滑点容忍度。
  - **Gas 倍数**：选择 Normal/Frontrun 等模式加速交易。
- **风控配置**：
  - **分批止盈**：可视化拖拽设置 4 个止盈档位。
  - **每日最大亏损**：触发熔断停止交易。
  - **最低安全分**：低于此分数的代币不买入。

## 4. 核心配置详解 (config.yaml)

虽然可以在 Web 界面修改大部分配置，但部分底层配置仍需在 `bsc_bot/config.yaml` 中调整：

### 4.1 网络节点 (Nodes)
为了保证交易速度，建议配置付费的高速 RPC 节点。
```yaml
network:
  nodes:
    primary: "wss://..."    # WebSocket 节点，用于监听新币
    execute:                # 执行节点列表，机器人会自动选择最快的节点
      - "https://..."
      - "https://..."
```

### 4.2 运行模式
```yaml
network:
  test_mode: true  # true=模拟模式（不花钱），false=实盘模式（慎用！）
```
**注意**：实盘模式下请确保私钥对应的钱包有足够的 BNB 作为 Gas 和本金。

## 5. 常见问题排查

**Q: 为什么日志显示 "RPC Connection Error"？**
A: 请检查网络连接，或在 `config.yaml` 中更换可用的 RPC 节点。建议使用科学上网工具。

**Q: 为什么发现了新币但没有买入？**
A: 可能原因：
1. **安全分过低**：代币未通过安全检测（如貔貅盘、流动性不足）。
2. **余额不足**：钱包 BNB 余额不足以支付买入金额 + Gas。
3. **测试模式**：当前处于测试模式，只会模拟买入。

**Q: 如何更新代码？**
A: 拉取最新代码后，建议重新运行 `pip install -r requirements.txt` 和 `npm install` 以确保依赖最新。
